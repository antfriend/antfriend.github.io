How to use TTE

There are two main ways to use TTE. Choose the path that fits your workflow.

Do 1) OR 2). NOT both at the same time.

1) In an IDE (the vibe-coder way)

Use your agent-assisted editor to steer the workflow directly.

  1. Clone, fork, copy, or unzip the TTE GitHub repo.
  2. Open it in your agent-assisted IDE (for example, Claude Code or VS Code with Codex).
  3. Edit TTE_PROMPT.md and describe what you want.
  4. Tell your agent to read the readme and go, or just say, “you got this.”

2) In Python (API-driven)

Use the Python entry point with an API adapter (the OpenAI API adapter is the only one implemented right now).

Toot Toot Networks

A TTN can use IP/telnet or Meshtastic or a connected hybrid of both.

The edge semantics of the network align with the semantic edges of the TTDB. This means an esp32/Arduino class device can contain a TTDB that lets it know about itself and everyone else it knows about on the network, and, if given the information, one node could also "know" (contain a sub graph) of everyone known to another node. They can all share sensor data, direct message, and broadcast... hypothetically. TTN deployment is Work in Progress.

How to Toot Toot Engineering (Bare bones python Runner)

Pull down the TTE GitHub repo and follow this Quick start.

Quick start

  • Pull down, clone, fork, copy, or unzip the GitHub repo.
  • Install dependencies: pip install -r requirements.txt or pip install openai
  • Set API key: export OPENAI_API_KEY="..." (macOS/Linux) or setx OPENAI_API_KEY "..." (Windows PowerShell)
  • Edit TTE_PROMPT.md and describe what you want.
  • Run agent: python tte_agent.py (or python3 tte_agent.py)
  • Run monitor: ./run_tte.sh, run_tte.bat, or .\run_tte.ps1

Notes

  • The agent reads and operates inside the repo workspace.
  • tte_monitor.py uses standard library, but needs tkinter.

Install tkinter

  • macOS: brew install python-tk
  • Ubuntu/Debian: sudo apt-get install python3-tk
  • Fedora: sudo dnf install python3-tkinter
  • Arch: sudo pacman -S tk
  • Windows: python.org installer with "tcl/tk and IDLE" enabled

Troubleshooting

  • "Missing OPENAI_API_KEY": confirm the env var in your shell.
  • Cannot import openai: reinstall deps or activate the right venv.
